Quickstart

Get StreamAPI running in your application in under 10 minutes. This guide walks you through authentication, your first API call, and handling a response — in Python and cURL.

Prerequisites

🔑

API Key

Sign up at streamapi.io and generate a key from your dashboard.

🐍

Python 3.8+

Or use cURL for language-agnostic examples throughout this guide.

📦

streamapi SDK

Install with pip install streamapi for Python examples.

🌐

HTTPS access

All StreamAPI requests go to api.streamapi.io/v1.

Authentication

StreamAPI uses API key authentication. Pass your key in the Authorization header with every request. Keys are prefixed with sk_ and scoped to your account.

⚠️

Keep your API key secret. Never expose it in client-side code, public repositories, or logs. Use environment variables to manage it in production.

Shell
# Set your key as an environment variable
export STREAMAPI_KEY="sk_live_your_api_key_here"

Install the SDK

Shell
pip install streamapi

Your First API Call

The following example creates a new stream and pushes an event to it. A stream is a named channel for real-time data. An event is a payload you push to that stream.

Python
import os
import streamapi

# Initialise the client
client = streamapi.Client(api_key=os.environ["STREAMAPI_KEY"])

# Create a stream
stream = client.streams.create(
    name="user-events",
    retention_days=7
)

print(stream.id)  # str_01HXYZ...

# Push an event to the stream
event = client.events.push(
    stream_id=stream.id,
    payload={
        "user_id": "u_789",
        "action": "signup",
        "plan": "pro"
    }
)

print(event.id)    # evt_01HXYZ...
print(event.status) # "delivered"
cURL
curl -X POST https://api.streamapi.io/v1/events \
  -H "Authorization: Bearer $STREAMAPI_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "stream_id": "str_01HXYZ",
    "payload": {
      "user_id": "u_789",
      "action": "signup",
      "plan": "pro"
    }
  }'

Understanding the Response

A successful event push returns a 201 Created response with the event object.

201 Created application/json 
{
  "id": "evt_01HXYZABC123",
  "stream_id": "str_01HXYZ",
  "status": "delivered",
  "payload": {
    "user_id": "u_789",
    "action": "signup",
    "plan": "pro"
  },
  "created_at": "2026-05-16T09:41:00Z",
  "delivered_at": "2026-05-16T09:41:00.042Z"
}
ℹ️

Delivery latency is typically under 50ms for events under 64KB. The delivered_at timestamp reflects when the event reached all active subscribers on that stream.

Error Handling

StreamAPI uses standard HTTP status codes. All errors return a JSON body with a code and message field.

StatusCodeMeaning
401unauthorizedAPI key missing or invalid
403forbiddenKey lacks permission for this resource
404stream_not_foundThe specified stream ID doesn't exist
422validation_errorRequest body failed validation
429rate_limitedExceeded your plan's request limit
500server_errorUnexpected error on our side — retries are safe
Python — Error handling
import streamapi
from streamapi.exceptions import StreamAPIError, RateLimitError

try:
    event = client.events.push(stream_id=stream.id, payload={...})
except RateLimitError:
    # Back off and retry — use exponential backoff in production
    time.sleep(1)
except StreamAPIError as e:
    print(e.code, e.message)

Rate Limits

Rate limits are applied per API key and vary by plan. Check your current usage in the dashboard.

  • Starter: 1,000 events/min, 100 streams
  • Pro: 10,000 events/min, unlimited streams
  • Enterprise: Custom limits — contact us

When rate limited, you'll receive a 429 response. The Retry-After header tells you how many seconds to wait before retrying.
